# Пункт 2 подробно: куда писать команды и как

Все команды пишутся **в терминале** в Cursor. Каждую команду — **отдельной строкой**, после каждой нажимаешь **Enter** и ждёшь, пока она выполнится. Потом вводишь следующую.

---

## Где писать

### 1. Открыть терминал в Cursor

- Внизу окна Cursor есть панель с вкладками. Одна из них — **Terminal** (Терминал).
- Нажми на неё — откроется чёрное (или тёмное) окно с текстом и мигающим курсором.
- Либо в меню сверху: **Terminal** → **New Terminal**.

В этом окне будет что-то вроде:

```
PS C:\Users\boiar\tower_bot>
```

или

```
PS C:\Users\boiar\...>
```

Это и есть место, куда нужно вводить команды. Курсор стоит в конце строки — туда и печатаешь.

---

## Как вводить: по одной команде

Команды **нельзя** вставлять все разом. Нужно так:

1. Напечатай **первую** команду (можно скопировать из инструкции).
2. Нажми **Enter**.
3. Дождись, пока в терминале появится новый текст (результат команды).
4. Напечатай **вторую** команду.
5. Снова **Enter**.
6. И так далее — **одна строка = одна команда = один Enter**.

Если вставить сразу несколько строк, выполнится только первая (или будет ошибка).

---

## Команды по порядку (пункт 2)

Ниже — команды **по одной**. Подставь **свой логин GitHub** и **название нового репозитория** (как ты его создал на GitHub). Пример: логин `maksimboyarkin468-ui`, репозиторий `tower-bot`.

---

### Команда 1: перейти в папку проекта

Напечатай **ровно** это (можно скопировать) и нажми Enter:

```
cd c:\Users\boiar\tower_bot
```

Что будет: в начале следующей строки путь поменяется на `PS C:\Users\boiar\tower_bot>`. Значит, ты в нужной папке.

---

### Команда 2: отвязать старый репозиторий

Напечатай и нажми Enter:

```
git remote remove origin
```

Обычно ничего не напишет — это нормально. Просто появится новая строка с `PS C:\Users\boiar\tower_bot>`.

---

### Команда 3: привязать новый репозиторий

Здесь нужно **подставить свои данные**. Вместо `ТВОЙ_ЛОГИН` — твой логин на GitHub, вместо `tower-bot` — название **нового** репозитория, который ты только что создал.

**Шаблон:**

```
git remote add origin https://github.com/ТВОЙ_ЛОГИН/tower-bot.git
```

**Пример** (если логин `maksimboyarkin468-ui` и репо называется `tower-bot`):

```
git remote add origin https://github.com/maksimboyarkin468-ui/tower-bot.git
```

Напечатай свою строку (со своим логином и именем репо) и нажми Enter. Ошибки быть не должно — снова просто новая строка.

---

### Команда 4: добавить все файлы

Напечатай и нажми Enter:

```
git add .
```

Точка в конце обязательна. Может появиться предупреждение про LF/CRLF — его можно игнорировать.

---

### Команда 5: сделать коммит (сохранить снимок)

Напечатай и нажми Enter:

```
git commit -m "Первый коммит: бот с постбэками 1win"
```

В ответ может быть что-то вроде: `[main xxxxx] Первый коммит... X files changed`. Это хорошо.

---

### Команда 6: отправить код на GitHub

Напечатай и нажми Enter:

```
git push -u origin main
```

Тут может быть так:

- **Спросит логин и пароль** — введи логин GitHub и в поле пароля **токен** (не пароль от аккаунта). Токен создаётся так: GitHub → Settings → Developer settings → Personal access tokens → Generate new token (classic), отметить **repo**.
- **В конце** должно появиться что-то вроде: `Writing objects: 100% ... done` и `main -> main`. Значит, код ушёл в новый репозиторий.

---

## Кратко: куда и как

| Вопрос | Ответ |
|--------|--------|
| Куда писать? | В **терминал** в Cursor (вкладка Terminal внизу или Terminal → New Terminal). |
| По одной команде или все подряд? | **По одной**: напечатал строку → Enter → дождался результата → следующая строка. |
| Что подставлять? | В команде с `git remote add origin` — свой **логин GitHub** и **название нового репозитория** вместо `ТВОЙ_ЛОГИН` и `tower-bot`. |

Если на каком-то шаге появится красный текст (ошибка) — скопируй эту строку и напиши мне, на какой команде это произошло.
