# Подробно: GitHub и как обновить бота на Railway

## Что у тебя уже есть

- **GitHub** — сайт, где хранится код проекта (как облако для кода).
- **Твой репозиторий** (проект на GitHub):
  - **Название репозитория:** `toweryuiop`
  - **Твой логин на GitHub:** `maksimboyarkin468-ui`
  - **Ссылка на репозиторий:** https://github.com/maksimboyarkin468-ui/toweryuiop

Твоя папка `tower_bot` на компьютере уже привязана к этому репозиторию. Когда ты делаешь **push** (отправку), код уезжает на GitHub, и Railway оттуда забирает обновления и перезапускает бота.

---

## Пошагово: что сделать

### Шаг 1. Открыть терминал в папке проекта

1. В Cursor открой папку `tower_bot` (если ещё не открыта).
2. Внизу экрана нажми **Terminal** (Терминал) или меню **View → Terminal**.
3. Должна открыться строка, куда можно вводить команды. В начале может быть что-то вроде `PS C:\Users\boiar\tower_bot>`.

Если путь не `tower_bot`, введи и нажми Enter:

```
cd c:\Users\boiar\tower_bot
```

---

### Шаг 2. Добавить все изменения в «корзину» Git

В терминале введи **ровно** эту строку и нажми Enter:

```
git add .
```

Точка в конце обязательна. Так ты говоришь Git: «все изменённые и новые файлы подготовь к отправке».

---

### Шаг 3. Сделать коммит (сохранить снимок с подписью)

Введи и нажми Enter:

```
git commit -m "Постбэки 1win, авто-проверка депозитов, фото"
```

Так ты создаёшь «коммит» — снимок текущего кода с коротким описанием. Ошибка вроде «nothing to commit» значит, что `git add .` не нашёл изменений (проверь, что ты в папке `tower_bot` и что файлы там менялись).

---

### Шаг 4. Отправить код на GitHub (push)

Введи и нажми Enter:

```
git push origin main
```

- **git push** — отправить коммиты на удалённый репозиторий.
- **origin** — имя удалённого репозитория (у тебя это GitHub).
- **main** — ветка, в которую отправляем.

После этого код уйдёт в репозиторий:  
**https://github.com/maksimboyarkin468-ui/toweryuiop**

---

### Если при `git push` просят логин и пароль

GitHub больше не принимает обычный пароль при push. Нужен **токен**:

1. Зайди на https://github.com — войди в аккаунт `maksimboyarkin468-ui`.
2. Профиль (иконка справа вверху) → **Settings**.
3. Слева внизу: **Developer settings** → **Personal access tokens** → **Tokens (classic)**.
4. **Generate new token (classic)**. Название любое (например, `railway-bot`), срок действия — по желанию, отметь право **repo**.
5. **Generate token** — скопируй токен (он показывается один раз).
6. В терминале при запросе:
   - **Username:** твой логин GitHub (`maksimboyarkin468-ui`).
   - **Password:** вставь **токен** (не пароль от аккаунта).

После успешного push в терминале будет что-то вроде: `Writing objects: 100% ... done`.

---

### Шаг 5. Railway подхватит обновление

1. Зайди на https://railway.app и открой свой проект (тот, что связан с GitHub).
2. Убедись, что проект подключён к репозиторию **maksimboyarkin468-ui/toweryuiop** (в настройках сервиса обычно видно репозиторий и ветку `main`).
3. После `git push` Railway сам запустит новый деплой (обычно через 1–2 минуты).
4. В проекте Railway открой вкладку **Deployments** — там появится новый деплой и статус (Building → Success).

Когда деплой станет зелёным (Success), бот уже работает с новым кодом.
